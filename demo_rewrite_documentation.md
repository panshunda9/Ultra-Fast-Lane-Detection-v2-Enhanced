# demo.py 重写说明文档

## 一、原代码说明 (第 1-121 行，已弃用)

### 原代码实现逻辑：

1. **输入模式**：依赖 DataLoader 读取测试集索引文件 (`.txt`)，从数据集中逐帧加载图片。
2. **可视化效果**：使用 `cv2.circle` 绘制离散绿色圆点标记车道线位置。
3. **输出形式**：将推理结果逐帧写入，最终生成 `.avi` 视频文件。

### 存在的不足：

- 无法直接推理本地单张图片或 MP4 视频，必须配置完整的数据集环境。
- 离散点视觉效果破碎，缺乏连贯性，不符合实际展示需求。
- 未包含图像裁剪逻辑，直接全图推理会导致模型视角错位，准确率下降。

### 重写方向：

改为支持本地文件实时推理，使用几何直线替代离散点，并加入视角裁剪对齐。

---

## 二、重写代码详细说明 (第 124 行起)

为满足本地演示与多样化输入需求，我对推理流程进行了全面重构：

### 1. 多模态输入支持 (第 219-283 行)

为实现这个效果，我新增加了两个文件：一个是images文件,用于存放我们即将推理的照片/视频；另一个是result文件，用于存放我们的推理结果

- **新增视频流推理**：通过 `cv2.VideoCapture` 直接读取 `.mp4`/`.avi` 等视频文件，实现逐帧实时推理与窗口播放 (`cv2.imshow`)，按 'q' 键可提前退出。
- **新增单图推理**：支持直接传入本地 `.jpg`/`.png` 路径，即时显示并保存结果）。

### 2. 可视化升级：几何直线拟合 (第 255-270 行 / 第 304-319 行)

- **弃用**：`cv2.circle` 离散点标记。
- **新增**：`np.polyfit` 一阶线性拟合，将检测点转换为连续几何直线。
- **新增**：自动截断逻辑，通过提取 `min_y`/`max_y` 限制直线范围，消除顶部多余交叉线。

### 3. 预处理修正：视角裁剪对齐 (第 245-246 行 / 第 294-295 行)

- **新增**：`img_tensor[:, :, -cfg.train_height:, :]` 切片操作。
- 切掉天空区域，仅保留底部路面，确保推理输入与模型训练分布一致。

### 4. 批量处理扩展 (独立脚本 `batch_infer.py`)

- **新增**：对文件夹的自动化批量推理，支持过滤非图片文件，结果统一归档保存。

### 5. 实时反馈与保存 (第 321-328 行)

- **新增**：`cv2.imshow` 实时窗口预览，配合 `cv2.waitKey` 实现交互控制。
- 自动按原文件名生成 `result_xxx.ext` 并保存至 `result` 目录。

## 三、使用示例

### 单张图片推理

python demo.py configs/tusimple_res18.py --test_model weights/tusimple_res18.pth --input_path images/你的照片名称.jpg（这时我们的案例图片放于images这个文件下）

### 视频推理

python demo.py configs/tusimple_res18.py --test_model weights/tusimple_res18.pth --input_path 你的视频名称.mp4(这时我们的案例视频放在项目的根目录下)

### 批量推理

python batch_infer.py configs/tusimple_res18.py --test_model weights/tusimple_res18.pth --input_path images/obj_train_data --test_work_dir result/obj_train_data_res（我们是将 images/obj_train_data下的多张照片进行批量推理，结果存于result/obj_train_data_res下）
